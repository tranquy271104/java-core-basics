## 1. Bẫy Tràn Số (Integer Overflow) - Kẻ Thù Số 1

Kiểu `int` trong Java chỉ lưu trữ được tối đa khoảng **2.14 tỷ** ($2 \times 10^9$). Nếu kết quả tính toán vượt qua mốc này, biến sẽ bị tràn và trả về số âm (gây lỗi Wrong Answer).

* **Quy tắc vàng:** Luôn check kỹ phần **Constraints (Ràng buộc)**. Nếu biến đầu vào $\ge 10^9$, các phép cộng nhiều số hoặc phép nhân chắc chắn sẽ gây tràn. Lúc này bắt buộc phải dùng kiểu `long`.
* **Mẹo ép kiểu nhanh với `1L`:** Thay vì phải đổi toàn bộ biến thành `long` từ đầu, có thể nhân với `1L` để ép Java tính toán toàn bộ biểu thức dưới dạng `long`.
```java
int x = 1000000;
int y = 1000000;
// System.out.println(x * y); -> SAI (Tràn số)
System.out.println(1L * x * y); // ĐÚNG

```



## 2. Xử Lý Phép Chia & Số Thập Phân (Type Casting)

Trong Java, chia hai số nguyên sẽ mặc định là **phép chia nguyên** (cắt bỏ hoàn toàn phần thập phân).

* **Để lấy kết quả số thực (thập phân):** Ép kiểu một trong hai biến sang `double` hoặc nhân với `1.0`.
```java
int a = 70, b = 30;
System.out.println((double) a / b); // Trả về 2.3333...
System.out.println(a * 1.0 / b);    // Cách viết gọn hơn, kết quả tương tự

```


* **Làm tròn số thập phân:** Dùng `System.out.printf` thay thế cho `setprecision` của C++.
```java
System.out.printf("%.2f\n", a * 1.0 / b); // In ra 2 số sau dấu phẩy và xuống dòng

```



## 3. Đặc Trưng Của Thư Viện `Math`

* **Nhóm hàm luôn trả về `double`:** `Math.pow`, `Math.sqrt`, `Math.cbrt`, `Math.ceil`, `Math.floor`.
* *Bẫy:* Khi in trực tiếp có thể dính đuôi `.0` (VD: `100.0`). Cần ép về `(long)` hoặc `(int)` để lấy phần nguyên sạch sẽ.
* *Ví dụ:* `System.out.println((long) Math.pow(x, y));`


* **Nhóm hàm "Vào gì ra nấy":** `Math.max`, `Math.min`, `Math.abs`. Truyền vào `int` thì trả về `int`, không cần ép kiểu.
* *Lưu ý:* `Math.max/min` của Java chỉ nhận **đúng 2 tham số**. Để tìm max của 3 hay 4 số, phải lồng nhau: `Math.max(Math.max(x, y), z)`.



## 4. Xử Lý Các Chữ Số Bằng Toán Học

Thay vì dùng String tốn tài nguyên và thời gian chạy, hãy dùng toán học để xử lý các chữ số:

* **Lấy N chữ số cuối:** Dùng phép chia lấy dư (`%`). Toán tử này còn tự động loại bỏ bẫy số `0` vô nghĩa ở đầu (VD: `05` sẽ tự thành `5`).
* Lấy 1 số cuối: `n % 10`
* Lấy 2 số cuối: `n % 100`


* **Xóa N chữ số cuối:** Dùng phép chia nguyên (`/`).
* Xóa 3 số cuối: `n / 1000`



## 5. Mẹo Vặt Khác

* **Đọc 1 ký tự (`char`):** `Scanner` không có hàm `nextChar()`. Dùng cú pháp: `char c = sc.next().charAt(0);`.
* **Công thức đếm số lượng:** Số lượng số nguyên trong đoạn từ $a$ đến $b$ (bao gồm cả 2 đầu mút) là: `b - a + 1`.
* **Khoảng trắng dư thừa:** `Scanner.nextInt()` tự động phớt lờ mọi khoảng trắng (dấu cách, tab, enter), không cần bận tâm việc các số nhập vào cách nhau bao nhiêu khoảng trống. Tuy nhiên, khi in ra **(Output)**, phải giống hệt từng khoảng trắng theo đề bài.





## 6. Kỹ Thuật Hoán Vị (Swap) Hai Biến
Khác với C++ có sẵn hàm `swap(a, b)`, đối với các kiểu dữ liệu nguyên thủy (primitive types) trong Java như `int`, `long`, `double`, bạn bắt buộc phải dùng **biến tạm (temporary variable)** để hoán đổi giá trị.
* **Công thức chuẩn:**
  ```java
  int tmp = a; 
  a = b; 
  b = tmp;

```

## 7. Kỹ Thuật Bóc Tách Số Thực

Khi cần chia tách một số thực (VD: `9.73`) thành phần nguyên (`9`) và phần thập phân (`0.73`):

* **Lấy phần nguyên:** Ép kiểu về số nguyên để tự động cắt bỏ đuôi thập phân.
* `int phanNguyen = (int) f;`


* **Lấy phần thập phân:** Lấy chính số ban đầu trừ đi phần nguyên vừa tìm được.
* `double phanThapPhan = f - phanNguyen;`



## 8. Quy Tắc Chọn Kiểu Số Thực: Luôn Ưu Tiên `double`

Trong lập trình thi đấu (Competitive Programming) và giải thuật, **hạn chế tối đa việc dùng `float**`.

* **Lý do:** `float` (32-bit) chỉ lưu trữ chính xác được khoảng 7 chữ số. Nếu đề bài cho số lớn (VD: `1000000.73` - gồm 8 chữ số), `float` sẽ làm tròn sai và gây ra lỗi Wrong Answer ở các test case lớn.
* **Giải pháp:** Luôn sử dụng `double` (64-bit). Kiểu dữ liệu này lưu trữ chính xác tới 15 chữ số, đảm bảo an toàn tuyệt đối cho mọi bài toán yêu cầu độ chính xác cao. Đọc dữ liệu bằng `sc.nextDouble()`.

## 9. Bẫy Tràn Số Ẩn Trong Các Công Thức Toán Học

Không chỉ các phép tính hiển nhiên trong yêu cầu đề bài mới gây tràn số, mà các **công thức hình học/toán học cơ bản** cũng là những cái bẫy nguy hiểm.

* **Ví dụ:** Bài toán tính diện tích hình chữ nhật với chiều dài, rộng $\le 10^6$.
* Phép tính `chu_vi = (dài + rộng) * 2` lớn nhất là $4 \cdot 10^6$, dùng `int` vẫn an toàn.
* Phép tính `diện_tích = dài * rộng` lớn nhất là $10^{12}$, chắc chắn làm tràn `int`. Bắt buộc phải nhân thêm `1L` hoặc dùng biến `long` ngay từ lúc nhập liệu.
